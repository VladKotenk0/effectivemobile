Как читать схему

  1. Микросервис фиксирует событие (корзина без действий, отмена заказа, акция) и публикует его в очередь.
  2. Notification Service забирает событие, проверяет настройки пользователя и формирует текст push.
  3. Push Gateway Service находит device token и отправляет уведомление через FCM или APNs.
  4. Пользователь видит push в мобильном приложении.
  5. Пунктирная стрелка: при установке приложение регистрирует device token в Push Gateway — без этого доставка невозможна.
